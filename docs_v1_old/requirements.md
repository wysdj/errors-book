# 错题管理器 — 需求规格说明书 (SRS)

> 版本：v1.2 | 日期：2026-05-24 | 作者：DJ + wysdj

---

## 1. 引言

### 1.1 项目背景
学生日常学习中会产生大量错题，传统手抄或截图保存方式效率低下：
- 无法快速检索
- 缺乏系统性分类
- 复习时缺少互动讲解

本项目旨在构建一个**拍照即录、智能分类、费曼复习**的错题管理系统。

### 1.2 目标用户
- **主要用户**：学生（录入错题、费曼讲解复习）

### 1.3 产品范围
- ✅ 拍照 → OCR → 自动分类 → 入库（支持印刷体 + 手写体）
- ✅ 多维度错题浏览和检索
- ✅ 费曼学习法复习（学生当老师讲解，系统评判）
- ✅ 统计分析仪表盘
- ❌ 不是在线题库（不自带题目数据）
- ❌ 不是社交平台

## 2. 功能需求

### FR-01 拍照录入

| 字段 | 描述 |
|------|------|
| 描述 | 用户通过摄像头拍摄或上传图片，系统自动识别题目内容 |
| 输入 | JPEG/PNG 图片，≤10MB |
| 处理 | 1) 图片预处理（去噪、增强） 2) OCR 文字提取（支持印刷体 + 手写体切换） 3) 智能分类 4) 知识点标注 |
| 输出 | 题目文本 + 学科 + 知识点标签 + 难度 + OCR 置信度 |
| 异常 | 识别失败 → 提示用户手动输入；手写识别低置信度时警告用户复核 |
| 备注 | 手写体识别准确率取决于字迹工整度（预估 70-85%），系统给出置信度供用户判断 |

### FR-02 手动录题

| 字段 | 描述 |
|------|------|
| 描述 | 用户手动输入题目文本和答案，选择分类 |
| 输入 | 题目文本、答案文本、学科、知识点 |
| 输出 | 结构化错题记录入库 |

### FR-03 错题浏览
- 列表视图：所有错题，支持分页
- 筛选：按学科、知识点、难度、日期、复习次数
- 搜索：关键词全文搜索
- 排序：按时间、错误次数、最近复习时间

### FR-04 错题详情
- 显示：原题图片 + OCR 文字 + 标准答案
- 操作：编辑、删除、标记"已掌握"
- 记录：添加笔记、标注错误原因

### FR-05 复习系统（核心）—— 费曼学习法

> 设计理念：学生扮演「教师」角色，用自己的语言讲解错题解法，系统评判讲解的完整性和正确性。
> 会做题 ≠ 真懂，能讲清楚才说明真正掌握。（费曼学习法）

#### FR-05.1 费曼讲解模式

学生可选择以下两种讲解方式之一：

##### FR-05.1a 自由讲解（文字 / 语音）
| 字段 | 描述 |
|------|------|
| 触发 | 学生选择一道错题 → 进入「自由讲解」 |
| 输入 | 学生用自然语言完整讲解题目（支持文字输入 + 语音输入，语音自动转文字） |
| 内容 | 题目类型、解题思路、关键步骤、注意事项 |
| 系统判定 | 4 维度检查：① 知识点覆盖完整性 ② 逻辑推导正确性 ③ 关键步骤是否遗漏 ④ 表达清晰度 |
| 输出 | ✅ 通过 → 标记「已掌握」；❌ 未通过 → 指出遗漏点 + 提示补充方向 |
| 循环 | 学生根据反馈改进讲解，重新提交 |

##### FR-05.1b 引导问答（系统问 → 学生答）
| 字段 | 描述 |
|------|------|
| 触发 | 学生选择一道错题 → 进入「引导问答」 |
| 流程 | 系统根据题目知识点，分解为若干步骤，逐个提问 |
| 示例 | Q1:「这道题考察的是什么知识点？」→ 学生答 → 判定 |
|      | Q2:「第一步应该怎么做？为什么？」→ 学生答 → 判定 |
|      | Q3:「这里需要注意什么易错点？」→ 学生答 → 判定 |
|      | ... |
| 判定 | 每轮回答即时评判（✅ 正确 / ⚠️ 不完整 / ❌ 错误），错误时给出引导提示 |
| 输出 | 全部通过 → 标记「已掌握」；中间卡住 → 可切换到自由提问求助 |
| 记录 | 每轮问答和判定存档，形成对话式复习记录 |

#### FR-05.2 自由提问模式
- 复习卡住时可随时向系统提问求助
- 系统不直接给答案，而是提供知识点引导提示
- 对话历史保存

### FR-06 统计分析
- 错题总数、学科分布（饼图）
- 每日/每周录入趋势（折线图）
- 知识点薄弱项排行
- 费曼讲解通过率变化曲线
- 最近 7 天复习统计

### FR-07 数据管理
- 导出：错题导出为 PDF/CSV
- 备份：SQLite 数据库备份
- 清理：批量删除已掌握错题

## 3. 非功能需求

### NFR-01 性能
- OCR 识别：印刷体 ≤ 2 秒，手写体 ≤ 4 秒
- API 响应：≤ 200ms (P95)
- 前端首屏加载：≤ 1.5s

### NFR-02 可用性
- 移动端适配（响应式布局）
- OCR 识别准确率：印刷体 ≥ 90%，手写体给出置信度
- 操作步骤 ≤ 3 步完成录题
- 语音识别：支持中文普通话，延迟 ≤ 1 秒

### NFR-03 安全性
- 本地运行，数据不出本机
- 数据库文件加密（可选）
- 无第三方数据上传

### NFR-04 可维护性
- 模块化设计，各服务独立部署
- 完整单元测试覆盖
- API 文档自动生成

## 4. 用户故事

| ID | 角色 | 行为 | 目的 |
|----|------|------|------|
| US01 | 学生 | 拍照/上传错题 | 快速录入，支持手写体 |
| US02 | 学生 | 查看错题列表 | 浏览和搜索所有错题 |
| US03 | 学生 | 按学科/知识点筛选 | 针对性复习 |
| US04 | 学生 | 用自己的话讲解错题 | 通过费曼学习法检验理解 |
| US05 | 学生 | 获取讲解反馈 | 发现知识漏洞并改进 |
| US06 | 学生 | 向系统提问求助 | 卡住时获得引导提示 |
| US07 | 学生 | 查看统计图表 | 了解薄弱环节和复习进度 |
| US08 | 学生 | 导出错题 | 打印或备份 |

## 5. 验收标准

| 编号 | 条件 |
|------|------|
| AC01 | 拍照 → 识别 → 分类 → 入库，全流程 ≤ 5 秒 |
| AC02 | 支持数学/物理/化学/英语，印刷体 ≥ 90% 准确率，手写体给出置信度 |
| AC03 | 错题筛选和搜索在 0.5 秒内返回结果 |
| AC04 | 费曼讲解判定至少检查 4 个维度（知识点、逻辑、步骤、表达） |
| AC05 | 统计数据图表正确展示 |
| AC06 | 所有 API 返回正确的 HTTP 状态码 |
| AC07 | 费曼讲解模式支持多轮改进循环 |
