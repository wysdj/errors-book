## 一、整体架构：手机怎么跟服务器通信的

```
┌─────────────────┐         HTTP 请求          ┌─────────────────────┐
│   你的手机       │ ──────────────────────────→  │   阿里云服务器        │
│   (PWA App)     │                              │   121.40.212.86      │
│                 │ ←────────────────────────── │                      │
│  HTML + CSS + JS│         JSON 数据            │  Flask (Python)      │
│  (前端/客户端)   │                              │  端口 8901           │
└─────────────────┘                              └─────────────────────┘
                                                        │
                                                        ├── PaddleOCR (被移除，改VL)
                                                        ├── DeepSeek API (AI分类/答案/复习)
                                                        ├── 通义千问 VL (图片识别)
                                                        └── SQLite (数据存储)
```

**核心概念**：前后端分离。前端只管"画界面"，后端只管"算数据"。二者通过 HTTP 请求通信。
前端发一个请求 → 后端处理 → 返回 JSON → 前端更新界面。

## 二、后端代码详解 (`ocr_server.py`)

### 2.1 Flask：Web 框架

Flask 是 Python 的 Web 服务器框架，核心就是"路由"：

```python
@app.route('/ocr', methods=['POST'])      # 定义路由：POST /ocr
def ocr():                                  # 处理函数
    f = request.files.get('image')         # 从请求里拿图片
    # ... 处理 ...
    return {'text': '识别结果'}             # 返回 JSON
```

每一个 `@app.route(...)` 就是一条"路"：手机访问这个 URL，Flask 就调用对应函数。

### 2.2 所有路由一览

| 路由 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 返回首页 HTML |
| `/ocr` | POST | 拍照识别题目（VL → AI分类 → 数学公式转KaTeX） |
| `/classify` | POST | 纯文字AI分类 |
| `/generate-answer` | POST | AI生成参考答案 |
| `/describe` | POST | 图片文字描述 |
| `/review/start` | POST | 开始费曼复习 |
| `/review/answer` | POST | 提交复习答案 |
| `/review/hint` | POST | 获取复习提示 |
| `/errors` | GET/POST | 错题列表/新增 |
| `/errors/<id>` | GET/PUT/DELETE | 单条错题 CRUD |
| `/errors/<id>/master` | POST | 切换掌握状态 |
| `/stats` | GET | 统计数据 |
| `/backup` | GET | 下载备份 zip |
| `/export/csv` | GET | 导出 CSV |

### 2.3 /ocr 路由：最核心的流程

```python
@app.route('/ocr', methods=['POST'])
def ocr():
    # 1. 接收图片
    f = request.files.get('image')
    p = f'images/{uuid4()}.jpg'
    f.save(p)

    # 2. VL模型提取文字（通义千问看图识字）
    text = describe_image_to_text(p)

    # 3. DeepSeek 把数学公式转成 KaTeX 格式
    text = ask_deepseek("转成KaTeX格式...\\n{text}")

    # 4. AI 自动分类（学科、章节、难度）
    ai = classify_by_ai(text)

    # 5. 返回给前端
    return {'text': text, 'ai': ai, 'image_path': p}
```

**为什么要数学转 KaTeX？**
因为 `<textarea>` 里无法渲染数学符号。KaTeX 把 `\begin{bmatrix}` 这样的代码
渲染成课本上的漂亮矩阵。DeepSeek 负责把 VL 识别的原始文字 `[1 2;3 4]` 转成
`\begin{bmatrix}1&2\\3&4\end{bmatrix}`，然后 KaTeX 渲染。

### 2.5 费曼复习的多轮对话原理

```
学生开始复习
    │
    ▼
POST /review/start  → DeepSeek 生成 3-5 个引导问题
    │                  Q1: "什么是矩阵乘法的维度规则？"
    ▼
学生回答 → POST /review/answer  → DeepSeek 判定 + 追问
    │         "两个矩阵A(m×n)和B(p×q)..."  → "正确！那具体到这道题..."
    ▼
学生卡住了 → POST /review/hint     → DeepSeek 给提示不给答案
    │                                    "想想A的行数和B的列数..."
    ▼
全部通过 → 自动标记"已掌握"
```

## 三、数据库 (`database.py`)

### 3.1 表结构

```sql
CREATE TABLE errors (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    question_text   TEXT,           -- 题目文字
    subject         TEXT,           -- 学科（数据结构/高等数学...）
    chapter         TEXT,           -- 章节
    topics          TEXT,           -- 知识点（逗号分隔）
    error_reason    TEXT,           -- 错误原因（分号分隔的标签）
    difficulty      TEXT,           -- 难度
    reference_answer TEXT,          -- 参考答案
    image_path      TEXT,           -- 图片路径
    mastered        INTEGER DEFAULT 0,  -- 0=未掌握, 1=已掌握
    reviewed_at     TEXT,           -- 最后复习时间
    created_at      TEXT            -- 录入时间
);

CREATE TABLE reviews (
    id        INTEGER PRIMARY KEY,
    error_id  INTEGER,             -- 关联错题ID
    questions TEXT,                 -- 复习问题(JSON)
    passed    INTEGER DEFAULT 0,   -- 是否通过
    score     INTEGER DEFAULT 0,   -- 得分
    created_at TEXT
);
```

### 3.2 SQLite 怎么用

```python
import sqlite3

# 连接数据库（文件不存在就自动创建）
db = sqlite3.connect('errors.db')

# 执行 SQL
db.execute("INSERT INTO errors (question_text) VALUES (?)", ("题目文字",))
db.commit()

# 查询
rows = db.execute("SELECT * FROM errors WHERE subject=?", ("数据结构",)).fetchall()
```

**一句话**：SQLite 就是存到硬盘上的一个文件 `errors.db`，用 SQL 语句操作，跟 MySQL 一样只是不需要装数据库软件。

## 五、PWA：网页变App的原理

```
四个文件配合：
manifest.json     → 告诉浏览器"我是App，这是名字和图标"
sw.js             → Service Worker: 缓存文件，离线也能打开
index.html        → <link rel="manifest"> 声明自己是PWA
icon-192/512.png  → App图标
```

**工作流程**：
1. 手机打开网站 → 浏览器读到 `manifest.json`
2. 浏览器注册 `sw.js`（后台缓存关键文件）
3. 浏览器菜单出现"添加到主屏幕"
4. 用户点添加 → 桌面出现图标
5. 打开图标 → 全屏模式（没有地址栏），用缓存文件秒开
