# 错题管理器 — 需求规格说明书 (SRS)

> 版本：v1.3 | 日期：2026-05-29 | 作者：wysdj + DJ（老师）

---

## 1. 项目背景

### 1.1 问题描述
整理错题非常耗时——大部分时间花在"抄题目"而非"思考题目"。手动分类也头疼：一道题属于哪个知识点、在考纲中的位置，学生自己容易判断不准。

目前用手机拍照存相册，但相册只能存图——不能识别文字、不能分类、不能复习验证。拍多了就是一堆找不到用不上的图片。

### 1.2 解决目标
1. **拍照录入**：拍一张照片，自动提取题目文字，支持手动补充答案/过程
2. **自动分类**：按 学科 → 章节 → 考纲知识点 三层分类，支持手动调整
3. **费曼复习**：系统以学生身份追问，用户以教师身份讲解解题过程
4. **统计薄弱点**：按学科、章节、知识点统计，找出最需要加强的地方

---

## 2. 功能需求

### FR-01 拍照录入

| 字段 | 内容 |
|------|------|
| 输入 | 用户拍照或上传图片（JPEG/PNG，≤10MB） |
| 处理 | 1) OCR 提取文字 2) 自动分类（学科→章节→知识点）3) 可手动补充答案和解题过程 4) 入库 |
| 输出 | 识别出的题目文字 + 分类结果 + 答案+过程（可编辑） |
| 异常 | 识别失败→提示重新上传或手动录入 |

### FR-02 错题分类

| 字段 | 内容 |
|------|------|
| 层级 | 学科 → 章节 → 考纲知识点（一道题可属于多个知识点） |
| 手动调整 | 系统分类错误时，用户可手动修改分类 |
| 浏览 | 按学科筛选、按知识点筛选、关键词搜索 |

### FR-03 费曼复习

| 字段 | 内容 |
|------|------|
| 方式 | 系统以学生身份一步步提问，用户以教师身份回答解题全过程 |
| 判定 | 3 个检查点：① 过程是否有错误 ② 知识点理解是否正确 ③ 解题思路是否流畅 |
| 反馈 | 讲错时系统指出具体问题点 |
| 输入 | 支持文字输入和语音输入 |

### FR-04 统计分析

| 统计项 | 说明 |
|--------|------|
| 错题总数 | 总计 |
| 学科分布 | 各学科错题数和占比（饼图） |
| 章节薄弱排行 | 哪个大章节错最多（柱状图） |
| 知识点薄弱排行 | 哪个知识点最薄弱 |
| 时间趋势 | 最近一周/一个月新增多少（折线图） |
| 复习统计 | 复习次数、通过率变化 |

---

## 3. 验收标准

| 编号 | 条件 |
|------|------|
| AC01 | 拍照 → 识别 → 分类 → 入库，≤ 5 秒 |
| AC02 | 分类支持手动调整 |
| AC03 | 费曼复习支持 ≥ 3 轮追问 |
| AC04 | 统计图表正确展示 |
| AC05 | 一道题可标记多个知识点 |
